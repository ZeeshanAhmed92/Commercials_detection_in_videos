Ad Detection Services API

A high-performance, asynchronous audio fingerprinting API. Features int64 database optimization, thread-pool scaling, vectorized hashing, and safe background cancellations.

⚙️ Pipeline Architecture

The application implements an optimized 3-stage asynchronous pipeline, driven by a concurrent semaphore manager.

Stage 1: Preprocessing

S1_preprocessing_aud.py runs FFmpeg subprocesses via a ThreadPoolExecutor. Audio is extracted to 16kHz Mono and standardizes loudnorm targets dynamically to prep for fingerprinting.

Stage 2: SQLite Fingerprinting

S2_fingerprint_db.py converts spectrogram peaks into vectorized int64 bitpacked keys instead of SHA-1. Insertions use SQLite WAL mode safely guarded by db_lock.

Stage 3: Advanced Detection

S3_scan_test_improved_latest.py partitions hashes to search concurrently. Boundary refinement uses Scipy fftconvolve natively in C, releasing the Python GIL.

Job Orchestrator

app.py and utils.py track detailed per-channel queues. A semaphore (MAX_OUTER_TASKS) prevents OOM crashes. Real-time cancellation abruptly halts queued tasks.

💡
Int64 Hashing vs SHA-1:
The database natively stores fingerprints using a bitwise formula: (f1 << 40) | (f2 << 20) | dt. This provides a 10-20x speedup over string-based SHA-1 lookup and drastically shrinks RAM footprint during large queries.

🔧 Configuration Context

Your config.py acts as the single source of truth for the entire pipeline. Constants are tuned for massive datasets:

# Tunable Execution Limits
MAX_OUTER_TASKS = 3            # Max concurrent channel/date combos
HASH_LOOKUP_CHUNKS = 4         # S3 parallel hash lookup threads
REFINE_WORKERS = 4             # Boundary refinement thread pool size

# Signal Processing Tuning
AMP_MIN_DB = -75
MIN_CONFIDENCE = 0.50          # Base hash-match score required
REFINE_SKIP_HIGH_SCORE = 0.95  # Score threshold to bypass cross-correlation

🔌 API Reference

POST /detect

Submits a new async job to run S1 → S2 → S3. Scans all files matching parameters.

Request Body

{
    "channel": ["NewsChannel1"],    // Required. String or Array.
    "date": ["20240115"],           // Optional. Auto-detects if omitted.
    "language": ["Hindi"],          // Optional. Ad-sample subfolder.
    "time": ["09", "10"],           // Optional. "09" targets "09_00_00.mp4"
    "specific_ads": ["pepsi_ad"]    // Optional. Limit to exact ad filenames.
}

Response 202 Accepted

{
    "status": "accepted",
    "job_id": "8f7b2c9a-...",
    "total_tasks": 2,
    "message": "Job queued. Poll GET /job/{job_id} for live status."
}
GET /job/<job_id>

Retrieves rich telemetry about a specific job, including sub-tasks (channels/dates).

Response 200 OK

{
    "job_id": "8f7b2c9a-...",
    "status": "running",
    "tasks_queued": 1,
    "tasks_running": 1,
    "tasks_done": 0,
    "elapsed_sec": 45.2,
    "output_file": "Outputs/API_Results/Detection_results.json",
    "channels": [
        {
            "channel": "NewsChannel1",
            "date": "20240115",
            "language": "Hindi",
            "status": "running",
            "segments": null,
            "elapsed_sec": 45.2,
            "error": null
        }
    ]
}
GET /jobs?status=running,queued

Returns an aggregated list of all jobs in memory. You can optionally filter by status.

Response 200 OK

{
    "total": 5,
    "jobs": [
        {
            "job_id": "8f7b2c9a-...",
            "status": "done",
            "tasks_done": 2,
            "segments_detected": 145
        }
    ]
}
POST /job/<job_id>/cancel

Requests immediate job cancellation. Queued items are instantly skipped. Running items execute to completion gracefully to prevent FFmpeg memory leaks, then mark themselves cancelled.

Response 202 Accepted

{
    "message": "Cancellation requested. Queued channels skipped immediately.",
    "job_id": "8f7b2c9a-...",
    "status": "cancelling"
}
